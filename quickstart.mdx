Let's get you from zero to a running app in about five minutes.

<Steps>
<Step title="Install Fymo">
```bash
pip install fymo
```
</Step>

<Step title="Create a project">
```bash
fymo new my-app
cd my-app
npm install
```

This scaffolds a new project for you and installs its frontend dependencies.
</Step>

<Step title="Run it">
```bash
fymo dev
```

Open `http://127.0.0.1:8000` in your browser. You should see your new app running.
</Step>
</Steps>

That's it, you have a real Fymo app running on your machine.

## Project structure

Here's what `fymo new` just set up for you:

```
my-app/
├── app/
│   ├── controllers/     # Python controllers
│   ├── templates/       # Svelte components
│   ├── remote/          # Remote functions (SvelteKit-style RPC)
│   ├── jobs/             # Background task registry
│   ├── broadcasts/       # SSE channel definitions
│   ├── lib/               # $lib/* alias target (TS/Svelte only)
│   └── support/           # Shared server-side utilities
├── dist/                 # Built output (generated by `fymo build`)
├── fymo.yml               # Project configuration
└── server.py               # Entry point
```

See [the app/ directory](/app-directory) to learn what belongs in each folder.

## A minimal page

Every page in Fymo pairs a Python controller with a Svelte template. Here's the smallest one you can write.

```python
# app/controllers/home.py
def getContext():
    return {
        "title": "Welcome to Fymo",
        "message": "Python backend, Svelte 5 frontend.",
    }
```

```svelte
<!-- app/templates/home/index.svelte -->
<script>
  let { title, message } = $props();
  let count = $state(0);
</script>

<h1>{title}</h1>
<p>{message}</p>
<button onclick={() => count++}>Count: {count}</button>
```

The controller returns data, and the template renders it. That's the whole loop, and everything else in Fymo builds on it.
